# pseudo-fable-blender — 使い方

[English](HOWTOUSE.md) | 日本語

導入後の日常運用。設計思想と導入手順は [README.ja.md](README.ja.md) を参照。このパックが効くのは、タスクが Blender での 3D モデリング(bpy スクリプト経由のヘッドレス、または Blender MCP)のときだけで、それ以外では黙っている。品質最優先の設計なので、「軽くでいい」と言わない限り **hero ティア**を前提にレンダーと改善パスを惜しみなく使う。

## 導入すると何が変わるか

- ジオメトリを作る前に、エージェントが**スペックブロック**を提示する: 対象物とアイデンティティ特徴、アートディレクション、品質ティア、ターゲット用途、実寸の数値、予算、計測可能な完了基準5〜10個。あなたには一度の拒否権がある。
- ビルドは**ゲート付きフェーズ**で進む: ブロックアウト(hero ではレンダー付きバリアント2〜3案から選択)→ シルエットゲート → フォルム → 3スケールディテール → マテリアル → ライト&カメラ。ブロックアウトゲートを通るまでディテール禁止。
- 意味のある変更のたびにプローブ画像をレンダーして実際に読み、「良さそう」ではなく具体的な欠点を名指しする。形は**クレイレンダー**で、トポロジーは**ワイヤーフレームレンダー**で、マテリアルはカラーマネジメント済みの**ビューティレンダー**でだけ判定する。
- 各ゲートには**ルーブリック採点**が付く — 6軸・1〜5点・各軸に証拠。点数の黙った後退は許されない。
- スペック合格は終わりではない: **excellence ループ**がレンダーをシニアアーティストとして批評し(パスごとに視点交代)、指摘を実装する — hero では「2パス連続で cosmetic のみ」になるまで。
- 「完了」は **QA ゲート**: 全基準にレンダーファイルかプローブ数値を対応付け、衛生チェック(非多様体=0、スケール適用、プローブ/リグの残骸なし)を通し、レンダー一式を報告に添付する。

## タスクの良い切り出し方

最初のメッセージに収穫の大きい5点を入れる:

> 「**2人掛けの公園ベンチ**を、**風化したミッドセンチュリー**(使い込まれた木+鋳鉄)で、**静止画レンダー用**に、幅**約1.5 m**、**hero 品質**で」

対象・アートディレクション・用途・サイズ・ティア。残りはデフォルトで足りる。参照画像がありエージェントが見られるなら、キックオフで渡す — 特にマテリアルとライティングでは、参照1枚が形容詞の段落に勝つ。ブロックアウト級でよければ「クイックで/draft で」と言う。言わなければ hero 級の工数(数十レンダー・複数改善パス)が設計どおり投入される。

## あなたの役割

| 場面 | やること |
|---|---|
| スペック提示 | 数値・アートディレクション・ティア・完了基準を検分する — これが QA ゲートの使う契約。拒否するならビューティレンダーの段階ではなく今。 |
| ブロックアウトバリアント(hero) | 灰色ブロックアウト2〜3案が並んだレンダーと、エージェントの選択+理由が届く。自由に覆してよい — ビルド全体で最も安い舵取りの瞬間。 |
| ブロックアウトゲート | 灰色プリミティブの4ビュー。見るのは比率だけ — この段階の不細工は予定どおり。 |
| フォルム/ディテール中 | パーツごとのレンダーと、ゲートごとのルーブリック採点が届くのが正常。レンダーなしの変更が来たらプローブセットを要求。 |
| excellence パス | 「シニアアーティスト批評」リスト(影響度順の5件)と修正が届く。早々に抜けたらパスログを要求 — hero は cosmetic のみの2連続パスが要る。 |
| 「完了」宣言 | カバレッジ表、証拠つきルーブリック、excellence ログ、レンダー一式(hero はターンテーブル+クローズアップ)が来るはず。証拠のない基準は未達成。 |
| 媒体の限界報告 | 「これはスカルプト級の有機的造形が要る」はパックが機能している証拠。4回目のリトライを押し付けず、選択肢(スタイライズ/参照提供/スコープ縮小)から選ぶ。 |

## プロトコル一覧

1. **SPEC**(`blender-spec`)— 依頼 → 数値+アートディレクション+ティア+完了基準。仮定は明記、質問はまとめて一度まで。
2. **シーン契約** — メートル法の実寸、命名計画、冪等ビルド、パーツの色分け。
3. **ブロックアウト → ゲート**(`blender-build-loop`)— hero はバリアント。4ビュー全てでシルエットが読めるまで先へ進まない。
4. **フォルム → ディテール** — 1パーツずつ、都度レンダー。3スケールのディテール。可視エッジ全部にベベル。トポロジーは `blender-topology`。
5. **マテリアル**(`blender-materials`)— まずクレイパス、それから PBR+ラフネス変化。判定はビューティレンダーで。
6. **ライト&カメラ**(`blender-light-camera`)— AgX/Filmic、3灯リグか HDRI、構図のあるカメラ。シーンは `blender-scene` でカメラ先行。
7. **VERIFY & EXCELLENCE**(`blender-verify`)— 証拠つきルーブリック、ティアの出口条件まで excellence パス、レンダー一式つき最終 QA。

## 舵取りフレーズ

- 「4ビューのプローブセットを見せて」(検証なしの変更が来たとき)
- 「コミットする前にバリアントを見せて」(hero のブロックアウト)
- 「まずブロックアウトゲート — ディテールはまだ」
- 「このレンダーの欠点を3つ挙げて」(「良さそう」が出たとき)
- 「クレイレンダーをちょうだい — そのマテリアルでは形が判定できない」
- 「ワイヤーフレームパスを — そのハイライトは揺れてる」
- 「データプローブを回して — 非多様体は? スケール適用は? tris は予算内?」
- 「ルーブリックを採点して、証拠も見せて」
- 「もう1周 excellence パスを — まだシニアアーティストの合格点じゃない」
- 「スペックの数値からずれてる — 座面高は 0.45 だったはず」
- 「これはスクリプトの表現力を超えてそう — 選択肢をちょうだい」(鉄則5の発動)

## 見るべき成果物

- 会話中の**スペックブロック**(long-task-state 導入時は `.claude/state/` にも)— 契約そのもの。
- `renders/` — イテレーションごとの `<prefix>_front|right|top|quarter.png`、`*_clay_*`・`*_wire_*`・`*_turn0..7`・`*_close_*` の各セット、ビューティショット。
- **ビルドスクリプト**(ヘッドレス時)— パラメータ化・再実行可能。.blend と並ぶ本命の成果物。
- 出力中の `PROBE {…}` 行 — オブジェクトごとの tris・寸法・非多様体数・scale_applied。
- ゲートごとの**ルーブリック採点**と **excellence パスのログ**(各パスが何を変えたか)。
- チェックポイント — ゲート通過ごとの `asset_p1_blockout.blend` 等。選ばれなかった案は `Variants_Rejected` コレクションに。

## 任意のフック層を入れている場合

- Blender 作業の完了報告は `[blender-qa: pass]`(完了宣言でない停止は `[blender-qa: n/a]`+一行の理由)で終わる。マーカーなしで停止しようとすると QA 指示つきで弾き返される — それは `stop-blender-qa` フックが機能している証拠であってエラーではない。
- headless の `blender --background` 実行のたびにプローブのリマインダー(「renders/ の新規画像をすべて Read して…」)が流れる — 「見ろ」というパックのナッジであって失敗ではない。
- 個別/全体の無効化は `PSEUDO_FABLE_BLENDER_DISABLE=qa,probe|all`。登録確認はセッション再起動後に `/hooks`。
- カバレッジ表とレンダーの裏付けがない `[blender-qa: pass]` は偽マーカー — 指摘してよい。マーカー契約は「真実でないマーカーを刷ること」を鉄則違反と定めている。

## 誤動作したら

- **レンダーせずに納品してくる** → 「見ていないモデルを信じない — プローブセットを見せて」。変更ごとレンダーのリズムは鉄則1。
- **狂った比率の上にディテールを盛る** → ブロックアウトゲートを発動。シルエットが落ちているのに載せたディテールは定義上、手戻り。
- **マテリアルと形を1枚のレンダーで同時に判定する** → クレイパスを要求。あの分離には理由がある。
- **指摘ゼロの「良さそう」が続く** → 欠点名指しルール: 2回連続で指摘ゼロは見ていない証拠。
- **ルーブリックの水増し**(バンディングが見えるのに4点)→ `blender-verify` のアンカー表を突きつける。採点には証拠が要り、軸は黙って後退できない。
- **hero なのに基準合格直後に停止** → 「スペック合格は完了ではない」— excellence パスとログを要求。
- **アウトライナーに `Cube.001` / 再実行で重複** → シーン契約違反。意味のある命名と冪等な再ビルドを要求。
- **bpy の AttributeError ループ** → 「`bpy.app.version` をプローブして、持ち主を `dir()` して — 2度目の当て推量はしない」。
- **平板・灰色・浮遊するレンダー** → カラーマネジメント+リグ+床面(`blender-light-camera`)。Standard 変換でビューティを判定しない。
- **有機的形状で無限リトライ** → 同一ターゲット3連敗で「リトライではなく選択肢」が発動する。そこに立ち戻らせる。
